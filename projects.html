Leveraging Stable Diffusion for Artificial Image Data Generation

This project explores the use of Stable Diffusion for generating artificial images to enhance classification models. The goal is to mitigate challenges such as small dataset sizes, imbalanced classes, and improper data labeling by creating synthetic training data.

Problem Statement

Acquiring high-quality labeled training data is often expensive and time-consuming. Our project investigates whether artificially generated images using Stable Diffusion can serve as a substitute for real-world datasets, particularly in training image classifiers for CIFAR-100.

Methods & Approach

  • Generated artificial images using Stable Diffusion based on CIFAR-100 class labels.
  • Scaled images from 512x512 down to 32x32 to match CIFAR-100 resolution.
  • Trained and tested classification models (ResNet50, SeresNet152, DenseNet201) on both real and artificial datasets.
  • Conducted hyperparameter tuning to optimize performance.

Key Findings & Results

  • Training on artificial data resulted in significantly higher classification errors (Top1 Error of 75.14% vs. 22.61% for real CIFAR-100 data).
  • Artificial images suffered from content mismatch, leading to poor generalization.
  • Increasing dataset size improved performance slightly but did not close the gap with real data.

Challenges & Insights

  • Stable Diffusion often generated unrealistic or cartoon-like images at lower resolutions.
  • Some class labels required additional descriptors to produce recognizable images.
  • Training models on synthetic data introduced biases that did not generalize well to real-world test sets.

Real-World Applications

Potential applications include data augmentation in medical imaging, adversarial training in cybersecurity, and synthetic dataset creation for machine learning research.



Data Mining Analysis

This project explores data mining techniques to analyze purchasing patterns and clustering methods in large datasets.

Problem Statement

The goal is to extract meaningful insights from transaction data, identify frequent itemsets, and apply clustering techniques to categorize data effectively.

Methods & Approach

  • Frequent itemset mining using support thresholds.
  • Apriori algorithm to analyze product co-occurrences.
  • K-means and Spectral Clustering for customer segmentation.
  • Feature extraction and Hashing Vectorizer for text-based analysis.

Key Findings & Results

  • Mineral water, eggs, spaghetti, French fries, and chocolate were the most frequently bought items together.
  • As support thresholds increased, fewer itemsets remained significant.
  • K-means clustering indicated that k=2 provided the best segmentation.
  • Increasing the number of features in Hashing Vectorizer improved classification accuracy.

Challenges & Insights

  • Choosing the optimal support threshold for frequent itemset mining.
  • Balancing feature size to optimize clustering without overfitting.
  • Interpreting spectral clustering results on high-dimensional data.

Real-World Applications

The techniques used in this project can be applied in retail analytics, customer segmentation, recommendation systems, and fraud detection.



BLaMo Design Document

The SLAM Business Layer Module (BLaMo) is a middleware component designed for the SLAM system, which assists municipal parks and recreation boards in managing sports leagues, tournaments, and scheduling.

Methodology:

The project was developed using an Object-Oriented (OO) design approach with a layered architecture. It consists of four main subsystems: Registration, Scheduling, Payment, and Notification. UML diagrams were created using LucidChart and Draw.io to illustrate class relationships and data flow.

Key Findings:

  • Utilizing a modular design improved system scalability and maintainability.
  • Generalizing League and Tournament into a single Competition class reduced redundancy.
  • Third-party services for messaging and payments improved security and reliability.

Challenges & Lessons Learned:

  • Balancing flexibility with predefined constraints in database storage.
  • Ensuring smooth integration between BLaMo, UI, and the database.
  • Optimizing performance while maintaining a robust notification and scheduling system.

Future Work:

Potential improvements include expanding payment options beyond credit/debit cards, enhancing real-time scheduling adjustments, and integrating machine learning for predictive analytics in game scheduling.
